# Getting Started

## Introduction

This package allows developers to use Proxyflare on any website deployed on the [Cloudflare Pages](https://developers.cloudflare.com/pages) platform.

## Install

Install `@flaregun-net/proxyflare-for-pages` using your preferred Javascript package manager.

```bash
npm install @flaregun-net/proxyflare-for-pages
# yarn add @flaregun-net/proxyflare-for-pages
```

## Set up

In your Cloudflare Pages project, create `functions/[[path]].ts` with the following boilerplate.

```ts
// functions/[[path]].ts
import proxyflare from "@flaregun-net/proxyflare-for-pages"

export const onRequest = [
  proxyflare({
    config: {
      routes: [],
    },
  }),
  // your other middleware
]
```

## Configure

Proxyflare is equipped to proxy incoming traffic to another service on your domain using a port proxy configuration, other services around the internet using a location proxy configuration, and simple redirect support.

The following section outlines a few basic configurations. To learn more, read the TODO full documentaion .

:::info Note

If you're using Typescript, Proxyflare for Pages comes with typed
configuration options.

:::

### Port Proxy

This port proxy configuration proxies sends traffic on `flaregun.net/api` to `example.com:8787` automatically appending any subpath requests to the destination. For example, a request to `flaregun.net/api/subpath` gets proxied to `flaregun.net:8787/subpath`.

```ts
// functions/[[path]].ts
import proxyflare from "@flaregun-net/proxyflare-for-pages"

export const onRequest = [
  proxyflare({
    config: {
      routes: [
        {
          from: "flaregun.net/example",
          to: "example.com",
          // optional request headers to pass along with the proxied request
          requestHeaders: [["Authorization", "Bearer token"]],
          // optional field to describe the configuration
          description: "proxy from example subpath to website",
        },
      ],
    },
  }),
]
```

:::info Note

<i>https:</i> is assumed if a protocol is not present in a configuration.

:::

### Location Proxy

This location proxy configuration sends incoming traffic on `flaregun.net/example` to `example.com` automatically appending any subpath requests to the destination. For example, a request to `flaregun.net/example/subpath` gets proxied to `example.com/subpath`.

```ts
// functions/[[path]].ts
import proxyflare from "@flaregun-net/proxyflare-for-pages"

export const onRequest = [
  proxyflare({
    config: {
      routes: [
        {
          from: "flaregun.net/api",
          to: "flaregun.net:8787/api",
        },
      ],
    },
  }),
]
```

### Redirect

This redirects incoming traffic on `flaregun.net/redirect` to `example.com` with a `redirectStatusCode` status code that defaults to `307`.

```ts
// functions/[[path]].ts
import proxyflare from "@flaregun-net/proxyflare-for-pages"

export const onRequest = [
  proxyflare({
    config: {
      routes: [
        {
          from: "flaregun.net/example",
          to: "example.com",
          statusCode: 301,
        },
      ],
    },
  }),
]
```

### Combining configurations

Proxyflare works with any number of route configurations. List your routes and Proxyflare will match the correct configuration.

```ts
// functions/[[path]].ts
import proxyflare from "@flaregun-net/proxyflare-for-pages"

export const onRequest = proxyflare({
  config: {
    routes: [
      {
        from: "flaregun.net/api",
        to: "flaregun.net:8787/api",
      },
      {
        from: "flaregun.net/example",
        to: "example.com",
        statusCode: 307
      },
      // more routes here
    ]
})
```
